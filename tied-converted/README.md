# TIED conversion output (sample)

This directory was generated by converting the monolithic STDD samples in `monolithic-samples/` to TIED format using the MCP conversion tools (or `node scripts/run-convert-stdd-to-tied.mjs`).

- **requirements.yaml** + **requirements/*.md** – requirement index and detail files
- **architecture-decisions.yaml** + **architecture-decisions/*.md** – architecture decision index and detail files
- **implementation-decisions.yaml** + **implementation-decisions/*.md** – implementation decision index and detail files

For real projects, copy the guide documents from the template repo (`requirements.template.md`, `architecture-decisions.template.md`, `implementation-decisions.template.md`) and point them to your YAML index and detail directories. Keep `semantic-tokens.yaml` updated with all REQ/ARCH/IMPL tokens (see AGENTS.md).
